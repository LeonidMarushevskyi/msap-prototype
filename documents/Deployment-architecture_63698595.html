Mississippi Agile Pool : Deployment architecture

Created by Leonid Marushevskiy, last modified by Monica Morales on Oct 10, 2016

Here is the proposed schema for this solution.
 
Workflow:

  1. Developers commit code to the public GitHub repository.
  2. Jenkins: Developer checks out the code, builds it, and runs unit tests.
  3. Sonar: Developer checks the code for quality, detects coverage percentage, etc.
  4. If all tests pass, Jenkins build Docker image and pushes it to the public Docker Hub.
  5. Amazon Cloud: pulls the image from the Hub and runs it.
  6. Application: sends monitoring data to the Zabbix server to collect statistics, fire alarms, provide graphs, etc.
