Mississippi Agile Pool : 02. Qualitative Research Results

Created by Pavel Khozhainov, last modified on Oct 12, 2016

Direct feedback result - see

03. Irina - October 11, 2016

Generic impression - UI decisions must be re-evaluated. Many minor but painful bugs.

Observation results

  1. Unclear function for Age Group selection on the Landing page
  2. Many minor UI bugs (unexpected popups, unexpected page refresh)
  3. Drop downs with multiple selection for ratings and age groups are unclear
  4. Visual ideas for facility presentation are not very clear - need a legend
  5. Automatic search apply is not clear - apply button expected

User needs based results (see needs here: 02. User Needs)

Parent - find the Child Care facility for own kid(s) - is possible to accomplish the task, but it is defined complex from the beginning

Foster Parent - work with the Caseworker to find child care for foster kid(s) (and own) - is possible to accomplish the task. Without provider role we have limited capabilities here. Also need an option to embed links to facility into the message texts.

Caseworker - find child care for children - is possible to accomplish the task. Need some advanced features like side by side comparison.

Step II (search result usage) needs - communication is possible. Some messenger improvements can be very useful.

Resume

  1. Landing page concept is good but must be simplified for anonymous user. Twitter, YouTube, Links to Login and Registration + few useful links. Basic search using home address only.
  2. Map navigation should be dynamic and aligned by result set. Use wants to see the map focused on search results.
  3. Overall impression is good
  4. For caseworker practice some integration should be implemented to import home location and age of the child(ren) from external caseworker module
