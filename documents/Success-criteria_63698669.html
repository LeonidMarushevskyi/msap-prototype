Mississippi Agile Pool : Success criteria

Created by Leonid Marushevskiy, last modified on Oct 10, 2016
#DeliverablesSuccess criteria
1.

Project Description and other Technical Documentation

Shall demonstrate an ability to meet the customer's need for a highly-skilled Agile team with proven experience

2.

Application prototype

  • Shall use modern technologies stack
  • High code quality
  • High test coverage
  • Responsive UI (mobile, tablet)
  • User-friendly UI
  • Compelling user experience
  • Shall delivered in a form of packed Git project with all branches and history
3.

Agile methodology artifacts

All stages of the prototype development process were documented in Confluence

4.

User-centric Design approach

  • All stages of the user-centric design approach were documented in Confluence
  • Modern techniques and tools were used
  • Interactive wireframe was implemented
  • Design mock-ups were implemented
  • All artifacts are stored in the GitHub repository
5.

Infrastructure

All infrastructure components are installed, configured, and publicly available for customer review

6.

Deployment and Continuous Integration

  • Prototype application deployed to IaaS, the PaaS public cloud provider
  • Ability to scale the application in the cloud
  • Automated continuous integration process is configured
  • Automated monitoring is available
