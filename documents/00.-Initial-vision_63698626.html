Mississippi Agile Pool : 00. Initial vision

Created by Pavel Khozhainov, last modified by Monica Morales on Oct 11, 2016

We have two major roles - caseworkers and parents.

Providers can be also added as separate role if we want to support social network style communications.

Caseworkers and parents will try to find a Child Care facility using the search filters and target location information.

Results will be presented as a map with marks related to facilities, plus a list of facilities.

Clicking on the mark will show  the details of the facility profile. Same functionality for clicking on the item in the 'found facilities' list.

Some advanced search criteria will proposed with additional attributes.

 

Facility attributes are:

Identification key (unique)

ProviderName

4 main attributes (we can use them for icon on the map):

LicenseType - LICENSED or UNLICENSED

ProviderType -

ProviderCapacity -  integer (up to 700 so 3 digits)

We can use the ProviderType as the icon type

The size of the icon will be ProviderCapacity  (3 sizes - small 0 to 10 (~3500), medium 10 to 100 (~1200), large 100+ (~250))

The color will be LicenseType (green licensed, red unlicensed)

QualityRating - add small stars around main icon

Option B: use quality rating as color and show license information as some special icon for UNLICENSED only

Location attributes

PhysicalZipCode

Unused attributes

We can ignore some attributes from initial data set

QualityRatingDescription - same as in enumeration code but string 5 to 0: {Excellent, Very Good, Good, Average, Low, Not Rated}

CountyNumber - we do not need county except when we want to group by this value

CountyName - just label for the code
