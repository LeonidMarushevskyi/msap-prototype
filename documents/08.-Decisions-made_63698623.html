Mississippi Agile Pool : 08. Decisions made

Created by Pavel Khozhainov, last modified by Monica Morales on Oct 11, 2016

Data

In order to provide a better user experience, we decided to extend child care provider data with full addresses.Use of full addresses provides the ability to map provider data for better visualization of information and comparison of providers based on an identified important criteria - distance. The data set provided by State of Mississippi was based on Texas data and real City/County names. At the same time, the real addresses of Child Care Centers (but not homes) are also available to the public. So we used Google API to generate some addresses.

Based on the caseworker interview data and additional data gathered from possible users (parents / case workers), we identified other factors users would like to use to evaluate child care providers.  The caseworker interviews provided agreement that the data could be extended and we have proposed the following additional data elements:

  • Openings available - randomly generated data. Most facilities have available slots.
  • Working hours - depends of Facility type we create some reasonable hours. Some facilities also receive Second Shift, Overnight And Weekend/Holiday hours.
  • Price - we use Child Care Aware data for year 2015 and generate normally distributed values rounded to $5 for all age groups supported by facility
  • Special needs support - some Child Care facilities randomly receive Special Need support ability
  • Number of complaints, visits, substantiated allegations - randomly generated data.

User Groups

There are four different user groups identified for the prototype - parents, foster parents, caseworkers, providers. Provider case is not supported in prototype due to limited time.

To currently address the short term need for input, the following process was used for identification of feedback groups.

  • Broad parent communication channels (discussion groups) with open membership
  • Used the same parent groups for foster parent feedback as the needs were similar and scenarios could be described easily
  • Caseworkers known to our identified interviewer and not to the team

  • Future effort would involve either third party service (such as Loop to identify potential audience) or working with the State to create the target user groups for feedback.. 

Features

Before prototyping, we received clear vision that we should add some communication feature (chat or messaging) to the application, based on user surveys.

Due to limited time we did not implement three side communication Foster Parent - Caseworker - Provider. We also did not implement Parent's review feature for Child Care Facilities. According to the development  process both features are placed to the Project backlog.

Some users showed interest in the side by side facility comparison - this feature goes to backlog.

Tools

We will use SurveyMonkey rather than EngagePoint Assist for gathering input from independent parties.

We also use Loop11 as advanced usability check tool

Other

From caseworker meeting we received information that some non-English language support will be needed. Spanish and Vietnamese are the options for user interface now.
